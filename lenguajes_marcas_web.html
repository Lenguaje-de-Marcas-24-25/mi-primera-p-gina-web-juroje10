Respuestas sobre HTML, XHTML y Organismos de Estándares

1. ¿Qué es el lenguaje HTML?

HTML, o HyperText Markup Language, es el lenguaje estándar usado para estructurar páginas web y su contenido. Mediante una serie de etiquetas y elementos, HTML organiza el texto, las imágenes, los enlaces y otros contenidos para que los navegadores puedan interpretarlos correctamente y presentarlos al usuario.

2. La relación de Tim Berners-Lee con HTML

Tim Berners-Lee es el creador de HTML, el cual desarrolló a fines de los años 80. Su intención era facilitar la conexión entre documentos en la red y proporcionar una forma estructurada de presentar la información en la web, algo que logró a través de HTML y los primeros navegadores web.

3. Versiones del lenguaje HTML

  • HTML 1.0: Primera versión, lanzada en 1991 con funciones básicas.
  • HTML 2.0: Estándar de 1995 que introdujo más etiquetas y funcionalidades.
  • HTML 3.2: Establecida en 1997, trajo mejoras en el diseño visual.
  • HTML 4.01: Versión de 1999 que popularizó el uso de CSS para estilizado.
  • HTML5: Versión más moderna de HTML, lanzada en 2014, que incluye soporte para multimedia y nuevas APIs.

4. ¿Qué es la W3C?

La World Wide Web Consortium (W3C) es una organización internacional que se encarga de desarrollar protocolos y directrices para la web, buscando mejorar su accesibilidad, compatibilidad y el desarrollo de estándares como HTML y CSS.

5. ¿Qué es el WHATWG?

El Web Hypertext Application Technology Working Group (WHATWG) es un grupo de trabajo creado en 2004 por empresas tecnológicas para centrarse en el desarrollo continuo de HTML y tecnologías relacionadas con la web, especialmente debido a desacuerdos con la W3C sobre cómo debía evolucionar HTML.

6. ¿Qué es un "Living Standard"?

Un Living Standard (estándar vivo) es un estándar que se actualiza y mejora de manera continua. HTML5 es un ejemplo de este tipo de estándar en manos de WHATWG, lo cual permite que evolucione de acuerdo con las nuevas necesidades y tecnologías de la web sin versiones fijas.

7. ¿Qué es el lenguaje XHTML?

XHTML, que significa eXtensible HyperText Markup Language, es una versión de HTML basada en las reglas del lenguaje XML. Esto hace que sea más riguroso en cuanto a la sintaxis y permite una mayor compatibilidad con aplicaciones de procesamiento de datos.

8. Versiones del lenguaje XHTML

  • XHTML 1.0: Publicado en 2000, adaptó HTML 4.01 a las normas XML.
  • XHTML 1.1: Versión modular lanzada en 2001, que permite más personalización y escalabilidad.
  • XHTML 2.0: Intento de estándar que finalmente fue abandonado en favor de HTML5.

9. Diferencias principales entre XHTML y HTML

  • Cierre de etiquetas: En XHTML, todas las etiquetas deben cerrarse, incluso las que en HTML pueden quedar abiertas (por ejemplo, <br />).
  • Uso de mayúsculas: XHTML obliga a que las etiquetas y atributos estén en minúsculas, algo opcional en HTML.
  • Atributos: En XHTML los atributos siempre deben ir entre comillas, mientras que en HTML es opcional.
  • Declaración XML: Los documentos XHTML suelen comenzar con una declaración XML (ej. <?xml version="1.0"?>), mientras que en HTML esto no es necesario.
